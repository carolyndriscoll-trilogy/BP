- Category 4: App-specific User-Interface (UI) Design 
  - Source 1
    - "The Psychology Behind Effective UX/UI Design: Understanding User Behavior" 
      - DOK1 - facts
        - fact 1 - cognitive load refers to the mental effort required to process information. To reduce this load, designers can employ "chunking," which involves grouping information into smaller, manageable units. For example, breaking down lengthy forms into multi-step processes can enhance user experience.
        - fact 2 - visual hierarchy guides users' attention by utilizing size, color, contrast, and placement. This concept is rooted in Gestalt Psychology, which explains how humans perceive patterns and group similar objects. By controlling visual hierarchy, designers can direct focus toward key actions or information.
        - fact 3 - emotional design operates on three levels: visceral (appearance), behavioral (usability), and reflective (personal meaning). Effective UX/UI design should consider all three levels to engage users emotionally, making the product more memorable and enjoyable
      - DOK2 - summary
        - summary point 1 - Gestalt principles guide perception because users naturally group similar elements, so applying similarity, proximity, and continuity helps designers structure interfaces that match how people visually process information
        - summary point 2 - When designs are too complex, users become overwhelmed. Using techniques like chunking information, minimizing unnecessary options (Hick’s Law), and clear visual hierarchy makes interaction smoother.
      - link to source [https://medium.com/@dariazaytseva07/the-psychology-behind-effective-ux-ui-design-understanding-user-behavior-a0abb5c0242e](https://medium.com/@dariazaytseva07/the-psychology-behind-effective-ux-ui-design-understanding-user-behavior-a0abb5c0242e)
  - Source 2
    - 7 Rules for Creating Gorgeous UI
      - DOK1 - facts
        - fact 1 - The vertical space between the menu items is fully twice the height of the text itself. If there is 12px font there should be just as much padding above and below it
        - fact 2 - Start with the harder problem of making the app beautiful and usable in every way, but without the aid of color. Add color last, and even then, only with purpose
        - fact 3 - Shadows are invaluable cues in telling the human brain what user interface elements we’re looking at 
      - DOK2 - summary
        - summary point 1 - key to believable and attractive UI design is mimicking how light behaves in the real world—specifically that “light comes from the sky.” This means elements should be shaded to reflect light from above, using subtle gradients and shadows to imply depth and layering. This technique helps users intuitively understand which elements are clickable or elevated, enhancing usability through visual cues.
        - summary point 2 - good UI design starts with layout and spacing—not color. Kennedy encourages designers to first build their interfaces in grayscale to focus on element hierarchy, white space, and alignment. Only after these fundamentals are nailed down should color be introduced, and even then, it should be purposeful (e.g., for emphasis or brand identity), not decorative. This method ensures clarity, structure, and consistency.
